# T2RSS 可视化管理面板

该目录是对原 Telegram 转发脚本的 Web 化重构版本，保留原有功能并提供后台管理。

## 完整保留的原功能

- 多源频道转发：基于 `CHANNEL_IDS` 执行。
- 按频道记录 `last_id`，支持断点续传（已升级为数据库存储）。
- 关键词黑名单过滤。
- 用户 ID 黑名单过滤（`MessageEntityMentionName`）。
- 夸克链接去重：
  - 目标频道预清理历史重复，
  - 当前批次内部去重，
  - 与目标频道历史链接比对去重。
- 锁文件防并发运行。
- 媒体下载 -> 发送 -> 临时文件清理。

## 新增能力

- 仪表盘：查看运行状态、断点进度、实时日志。
- 配置管理：在后台编辑并持久化保存环境变量。
- 会话管理：上传/删除 `t2rss.session`（上传任意 `.session` 后会自动统一命名）。
- 备份管理：创建、下载、删除系统数据备份。
- 备份恢复：支持从已有备份恢复，或上传 ZIP 备份恢复。
- 自动运行：支持后台定时自动触发。
- 管理员安全登录：默认开启登录校验与防爆破锁定。
- 断点管理面板：支持 `last_id` 的创建、查看、修改、删除。
- 保存 `CHANNEL_IDS` 时自动补齐断点：新 CID 会自动写入数据库断点，默认 `last_id=0`。
- CID 必经流程：新增频道前必须先解析 CID，再写入 `CHANNEL_IDS`。
- 测试模式开关：开启后仅模拟流程，不真实转发，不更新断点，不删除目标重复消息。
- 运行总超时：支持 `PANEL_TOTAL_TIMEOUT_SECONDS`（默认 600 秒），超时自动中止。
- 首页强制中止：任务运行中可一键强制中止当前转发任务。
- 首页实时日志：仪表盘实时拉取运行日志，替代最近运行记录表格。
- 实时日志支持“清空日志”按钮。
- 系统时间统一按上海时区（Asia/Shanghai）展示与记录。
- 日志输出包含 Emoji 阶段提示（如 🚀/📥/🧹/✅/⚠️）。

## 页面分组

- **初始化接入**：管理员与安全 + Telegram 接入参数 + 会话管理。
- **转发设置**：转发通道配置 + 过滤与去重策略 + 断点管理 + CID 解析工具。
- **计划与备份**：调度与自动运行 + 备份创建/恢复。

## last_id 兼容规则

- 旧版 `last_id` 文件目录规则保持不变：
  - 文件名（不含 `.txt`）就是频道 CID
  - 文件内容数值就是该 CID 对应的 `last_id`
- 新版启动时会自动把上述规则迁移到数据库表 `channel_last_id`。

## 持久化数据目录

所有持久化数据都在 `./data`（容器内挂载到 `/app/data`）：

- `data/config.env`：由后台管理页面保存的配置。
- `data/session/t2rss.session`：Telegram 会话文件。
- `data/panel.db`：运行历史、登录防爆破、频道断点（`channel_last_id`）数据库。
- `data/state/forwarder.lock`：运行锁文件。
- `data/state/downloads/`：媒体临时目录。
- `data/backups/*.zip`：备份压缩包。

## Docker 启动

```bash
docker compose up -d --build
```

打开：`http://localhost:8080`

## 首次使用流程

1. 打开 **初始化接入** 页面，填写管理员与 Telegram 接入参数。
2. 在 **初始化接入** 页面上传或创建会话文件。
3. 打开 **转发设置** 页面，配置目标频道、源频道、过滤与去重策略。
4. 在 **转发设置** 页面使用 CID 解析工具并写入 `CHANNEL_IDS`。
5. 在 **转发设置** 页面按需创建或修改各频道 `last_id`。
6. 打开 **计划与备份** 页面配置自动运行与超时策略。
7. 到 **仪表盘** 页面点击 **立即执行转发**。

会话文件准备方式：

   - 方式 A：在 **初始化接入** 页面上传任意 `.session` 文件（系统自动保存为 `t2rss.session`）。
   - 方式 B：在容器内交互创建：

```bash
docker exec -it t2rss-web-panel python tools/create_session.py
```

## 管理员登录与防爆破

- 服务启动时会自动检查管理员配置。
- 若未配置管理员密码，会自动生成随机初始密码并写入容器日志，请登录后立即修改。
- 登录失败达到阈值后会触发临时锁定（默认 5 次失败 / 10 分钟窗口 / 锁定 15 分钟）。
- 可在 **配置管理** 页面调整以下安全参数：
  - `PANEL_LOGIN_MAX_FAILURES`
  - `PANEL_LOGIN_WINDOW_SECONDS`
  - `PANEL_LOGIN_LOCK_SECONDS`
- `PANEL_SESSION_SECRET` 首次运行自动生成，默认不提供网页修改入口。
- 如需修改会话密钥，请使用 `docker exec` 进入容器编辑 `/app/data/config.env`，然后重启容器。

## 备份说明

在 **计划与备份** 页面可执行：

- 创建备份
- 下载备份
- 删除旧备份
- 从已有备份恢复
- 上传 ZIP 备份并恢复

恢复会先自动创建一份 `pre_restore_auto_*.zip` 回滚备份，再覆盖当前 `data/`（保留 `backups` 目录）。
